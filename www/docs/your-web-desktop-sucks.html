Your Web Desktop Sucks

The funny thing about web desktops is that they really attract crackpots. There seems to be a theory of everything aspect to web desktop development. But let's back up for a second, to circa 1980's. The only reason why desktop GUI's came into existence was because there was already some underlying operating system in place, accessible through a command-line interface. Then the whole "windows" concept came along. There was the X Window system (and I'm pretty sure something else called "windows" eventually came around, too).

So people have fun building all different kinds of websites. One of the kinds they might want to build might have some resemblance to a computer's desktop interface. But what is the chance that anyone will have any actual, real-world use for such a website? Well, because there is not actually any kind of a well-oiled machine that underlies the interface, then the chance is as close to nil as you can get.

I would say that Linux on the Web has a fairly well-oiled machine at its core. It works pretty good, methinks. Try it and see.

The main web-based desktop that you will find via search in 2024 is OS.js. Anders Evenrud spent quite a bit of time responding to people's issues on the OS.js messaging forum. That is "spent" as in past tense. It's been well over a year, now, since his most recent messages. And now there's another project that's been buzzing about for the past 2-3 years. The thing about that other one is that the developer is extremely active on message forums in promotion of his project. And especially on Hacker News.

This new developer has a lot of fun building his desktop and wants to keep doing it for, like, ever and ever. He really wants to fool you into thinking that you are using a native desktop interface. But it really sucks because the underlying system sucks. He just wants to keep piling more and more features into it. Try downloading the github repo for it, and you'll be waiting awhile (it's nearly 200MB, versus less than 1MB for mine!). He doesn't seem to care very much about the concept of a holistic, integrated system. He really seems to like the fact that there are long lists of neat sounding applications that can be invoked via start menu or the double-click of a desktop icon. Maybe he'll change eventually, and get more serious about the underlying system. But as it stands now, he just seems to have a lot of fun with the creation of appearances. I'm not saying that there's anything wrong with that in itself, but I am saying that there's a whole hell of a lot more you can do with the concept of web-based desktops than deceptive flashiness.

So if you search for "web based desktop" you will see that new one, a few results down from OS.js. But if you search instead for "web based operating system", you will find mine in that same position. The game is on!

Within the past year, there was a Hacker News thread about OS.js. The other guy that I'm obliquely referring to, literally, I think it's safe to say, hijacked that thread. He was responsible for 1 out of almost every 8 comments on it! So now let me be very blunt: Your web desktop sucks.

So let's hash this thing out, Hacker News. Enough with all of the "isn't that so neat looking" types of comments, as regards highly unorthodox and sophisticated kinds of web interfaces! I mean we are all painfully aware, by now, of the infinity of possibilities that modern web technologies give us. All I'm trying to say here is that I think we can start talking very deeply, now, about actual systems rather than the glittery interfaces that merely allow us to access them.
